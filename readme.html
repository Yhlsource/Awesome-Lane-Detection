SC-Lane 在 HeightLane 基础上提出斜率自适应和时序一致的高度估计框架。Slope-Aware Adaptive Feature (SAA) 模块：对每个多斜率锚点采样的 BEV 特征，使用轻量 CNN 从骨干图像特征预测逐像素锚点权重 α_a（softmax 归一化），动态加权融合不同斜率特征，替代等权拼接。锚点权重同时作为粗略高度置信度图，投影回 2D 图像空间与地面分割掩码计算 IoU 损失引导学习。Height Consistency Module (HCM)：利用自车运动补偿将 t-1 帧高度图变换到 t 帧道路坐标系，对重叠区域施加 L1 一致性损失（训练时两帧，推理仅需单帧无额外开销）。OpenLane F1 64.3%，弯道 70.7%、合流分道 63.0%、交叉路口 56.1% 均为场景最佳。高度估计 MAE 0.176m、Acc@0.1m 50.7% 全面超越基线。

GLane3D 有 Lite(ResNet-18, F1 61.5%, 62.2 FPS)、Base(ResNet-50, F1 63.9%)、Large(Swin-B, F1 66.0%) 三个变体。核心流程：检测3D关键点 → 预测有向连接 → Dijkstra提取车道线。自定义BEV几何解决近处稀疏/远处密集问题（透视投影导致均匀BEV网格投影到FV后图像间距与Z²成反比）。跨数据集（OpenLane→Apollo）F1保留率 89%（LATR 仅 55%），Bottom-up 方法天然泛化更强。GLane3D-Lite 是唯一 60+ FPS 的3D车道检测方案。

精度对比与SOTA分析：

  • OpenLane 最佳 F1：GLane3D 66.0%、SC-Lane 64.3%、Chen et al. 63.8%、HeightLane 62.7%、GroupLane 61.0%、LaneCPP 60.3%、BEV-LaneDet 58.4%、SPG3D 53.7%
  • 开创性工作：3D-LaneNet (ICCV 2019) 首个端到端3D多车道检测（F1 44.1%）；Gen-LaneNet (ECCV 2020) 泛化且可扩展
  • 纯单目SOTA：GLane3D (arXiv 2025) Bottom-up 关键点图新范式，精度和泛化性全面超越传统 Top-down 方法
  • 斜率自适应：SC-Lane (arXiv 2025) 动态融合多斜率高度特征 + 时序一致性，超越 HeightLane +1.6 F1
  • 效率优化：SPG3D (ICCV 2023) 稀疏点替代稠密对应，显存减半速度翻倍，F1 +3.2（vs PersFormer）
  • BEV转换创新：BEV-LaneDet 使用 VPN；GroupLane 使用 LSS；GLane3D 自定义BEV几何

Depth3DLane 提出双路径框架融合自监督单目深度估计与 3D 车道线检测。FV 路径（ResNet-18）提取语义特征；BEV 路径利用自监督深度网络（Lite-Mono 架构，冻结权重）预测密集深度图，反投影生成伪点云 P_t ∈ R^{HW×8}（含3D坐标+颜色+uv），经 PointPillars + ResNet-18 编码为 BEV 特征。3D 锚点分别投影到 FV 和 BEV 空间采样特征后拼接送入预测头。提出逐片段拟合（per-segment fitting）方法从逐帧学习的焦距中拟合稳定内参，使框架可在无相机标定的场景（如众包HD地图）中应用。OpenLane-1000（720×960）F1 56.6%，OpenLane-300（320×480）F1 64.3%（vs LATR 63.8%）。仅 26.9M 参数。

GTA-Net 通过 TGEM 模块从多帧运动中构建 Cost Volume 恢复深度（无显式BEV变换，类似LATR的query-based范式），通过 FiLM 调制将几何信息注入2D特征；TIQG 模块裁剪放大当前帧合成伪未来帧发现远处车道线，聚合过去+现在+伪未来三组查询。

精度对比与SOTA分析：

  • OpenLane 最佳 F1：Depth3DLane 64.3%（OpenLane-300）、PVALane 63.4%、GTA-Net 62.4%（时序）、LATR 61.9%、Freq-3DLane 59.7%、BézierFormer 56.43%、DecoupleLane 51.2%
  • 自监督深度融合：Depth3DLane (arXiv 2025) 双路径框架，无需GT深度和相机标定，适用于众包场景
  • 时序增强：GTA-Net (arXiv 2025) 引入多帧时序Cost Volume，纯单目3D时序方向首个代表
  • 曲线建模：BézierFormer (arXiv 2024) 贝塞尔曲线注意力统一2D/3D，F1 56.43%（151 FPS）；DecoupleLane (AAAI 2024) 曲线+路面解耦，CULane 2D mF1 56.32% SOTA
  • 新方向：Freq-3DLane (arXiv 2025) 引入频域分析提升3D车道检测

AurigaNet 基于 YOLOP/HybridNets 改进的端到端多任务网络。共享编码器（CSPDarknet + SPPF + FPN）+ 三个解码头：YOLOv5风格目标检测头、车道线分割头、可行驶区域实例分割头（二值分割 + 特征嵌入分支，引入可变形卷积使感受野对齐实例边界）。判别损失（discriminative loss）实现端到端实例分割无需后处理聚类。BDD100K：车道 IoU 60.8%（超第二名>30%）、Acc 98.77%、可行驶区域 IoU 85.2%、实例 mAP50 87.25%、mAP@0.5:0.95 47.6%。9.09M 参数，RTX 4080 上 217 FPS，Jetson Orin NX 上 5.077 FPS。

PoseDriver 基于 OpenPifPaf 的 bottom-up 统一框架，将车道线重新定义为骨架检测问题（固定数量等间距关键点）。使用 CIF 做关键点检测 + CAF 做部件关联。选用无 BN 的骨干（ConvNeXt, Swin）避免多域偏移。一个模型同时检测多类别骨架，多任务权重可迁移到未见类别。

BDD100K 评测协议注意：
车道线标签宽度：Lane IoU 受标签宽度影响极大。本表中 #2-#8 使用原始仓库 2-pixel 宽度标签评测（IoU 26-34%），#11 RMT-PPAD 使用 8-pixel 膨胀标签评测（IoU 44-57%）。#12 AurigaNet 的 Lane IoU 60.8% 声称比 HybridNets 提升 29.2%，但需确认标签宽度是否一致。RMT-PPAD 论文在同一8-pixel评测下：YOLOP IoU=44.0、HybridNet IoU=52.0、YOLOPv2 IoU 未报告。两种评测均为有效标准，不可直接横向比较。

十一、发展趋势

  1. 2D → 3D → 拓扑：从像素级向3D空间建模和语义拓扑推理演进
  2. 单帧 → 时序：视频方法利用帧间一致性提升 1-3% F1；3D领域 GTA-Net 引入时序几何增强
  3. Top-down → Bottom-up：GLane3D（BEV-based）关键点图新范式，跨数据集泛化保留率 89%（传统 Top-down 仅 55%），纯单目 F1 达 66.0%
  4. 单模态 → 多模态：Camera+LiDAR 联合（DV-3DLane 66.8%），但纯单目（GLane3D 66.0%）已逼近多模态，差距仅 0.8%
  5. 感知 → 建图：在线 HD 地图构建是论文数量最多的方向（36篇），2024年爆发式增长，mAP从HDMapNet 26.5%→MapTracker 74.7%
  6. 统一化：Lane2Seq 用序列生成统一多种车道表示
  7. 自/弱监督：减少标注依赖（WS-3D-Lane、CLLD、LaneCorrect）是重要探索方向
  8. 即插即用增强：EP-head + PL-attention 不改架构即可提升 +2.8~4.4 F1，发现并修复现有 sparse 表示的端点截断缺陷
  9. 斜率自适应高度估计：SC-Lane (arXiv 2025) 动态融合多斜率特征 + 时序一致性，F1 64.3%，超越固定斜率方法
  10. 自监督深度赋能3D检测：Depth3DLane (arXiv 2025) 双路径融合自监督深度，无需相机标定，适用于众包场景
  11. 2D先验增强拓扑：Topo2D (arXiv 2024) 利用2D高召回率先验初始化3D查询，OLS 44.5% 拓扑推理最佳
  12. MoE赋能端到端ADS：ExpertAD (AAAI 2026) PA感知适配器 + MoSE稀疏专家，碰撞率-20%、延迟-25%
  13. 端到端实例分割：AurigaNet (arXiv 2026) 判别损失+可变形卷积，车道IoU 60.8%（超第二名>30%），217 FPS
